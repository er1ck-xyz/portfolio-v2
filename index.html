Sobre mim

Comecei explorando uma ferramenta chamada Cheat Engine, usada até hoje para entender e modificar valores em jogos, como dinheiro, vida ou habilidades. No início, tudo era pura curiosidade: queria descobrir como os jogos funcionavam por dentro e como certos recursos eram manipulados.

Para aprender, assisti a diversos vídeos tutoriais, li fóruns e acompanhei outros entusiastas, sempre com foco em entender os conceitos por trás das modificações, e não apenas em usar cheats prontos. Cada experiência me ensinava mais sobre memória, lógica de programação e sistemas de jogo, despertando meu interesse em criar minhas próprias soluções do zero.

Com o tempo, comecei a estudar C++ e Assembly explorando conceitos avançados de programação e lógica de sistemas. No entanto, não levei esses estudos adiante de forma contínua naquela época, mas a experiência me deu uma base importante e despertou ainda mais minha paixão por tecnologia.

Essa fase foi fundamental para minha trajetória, mostrando que curiosidade, prática e estudo são a chave para evoluir como programador e solucionador de problemas complexos.

Resumo / O que é necessário para começar

A área de exploração de jogos e engenharia reversa exige uma combinação de habilidades teóricas e práticas. Para iniciar, é fundamental entender os conceitos básicos de sistemas de computador, memória e lógica de programação, pois eles são a base para qualquer modificação segura e eficiente. caso queira se aprofundar, aqui estão algumas habilidades e conhecimentos essenciais: leia mais sobre clicando aqui

Linguagens de Programação

  • C++: A linguagem mais utilizada para criar ferramentas complexas de modificação, injeção de código e desenvolvimento de drivers. Permite controle direto sobre memória, threads e recursos do sistema.
  • Assembly: Essencial para entender como os jogos e sistemas operacionais funcionam em baixo nível, identificar instruções, estruturas de memória e pontos de hook.
  • Python / C#: Úteis para criar scripts auxiliares, análises de memória ou interfaces gráficas para suas ferramentas.

Competências e Conhecimentos

  • Engenharia Reversa: Saber como desmontar, analisar e compreender binários de jogos, entender estruturas de dados e encontrar padrões de memória.
  • Manipulação de memória: Aprender a identificar valores dinâmicos, ponteiros e estruturas de dados de runtime.
  • Criação de drivers de kernel: Para modificar ou ler informações de processos de forma segura, evitando limitações do user-mode (como anti-cheats).
  • Assinatura de drivers: Para que os drivers possam ser carregados no Windows sem triggers de segurança ou anti-cheats, é necessário obter certificados digitais válidos.

Ferramentas e Recursos

  • Debuggers: Softwares como x64dbg ou OllyDbg para análise de executáveis.
  • Dumpers: Para localizar endereços dinâmicos em engines de jogos como Unreal Engine (ex: GNames e GWorld).
  • Cheat Engine: Excelente para aprender conceitos de memória, ponteiros e valores estáticos/dinâmicos.
  • Fóruns e Tutoriais: Comunidades de engenharia reversa e hacking ético, onde é possível aprender com projetos reais e exemplos.
  • IDA Pro / IDA Free: Uma das ferramentas mais poderosas para engenharia reversa, permite analisar código assembly, mapear funções, identificar chamadas importantes e compreender a estrutura do binário.
